# 城市选择页面重设计

## 概述

重新设计首页左上角城市选择入口的 `CityPickerScreen` 页面，支持三级行政区划选择（省/自治区/直辖市 -> 地级市 -> 区/县），新增 GPS 自动定位、搜索、最近使用记录、逐级下钻浏览等功能。

## 当前问题

- 仅支持 34 个大城市，无法覆盖中小城市和区县
- 无搜索功能
- 无 GPS 自动定位
- 无最近使用记录
- 视觉设计单调

## 设计方案

### 页面形式

全屏 Scaffold 页面，保持当前 `context.push('/city-picker')` 的导航方式不变。

### 页面结构（从上到下）

#### 1. 顶部导航栏

- 返回按钮 + 标题「选择城市」
- 保持当前 `PageTitleBar` 组件

#### 2. GPS 定位卡片

- 渐变背景（accent 色），圆角 14，带阴影
- 显示 GPS 定位到的省/市/区完整路径
- 「使用此位置」按钮
- 三种状态：定位中（loading）、定位成功、定位失败（可点击重试）
- 使用 `LocationService.currentPosition()` + Haversine 距离匹配最近的区县

#### 3. 搜索框

- 支持汉字和拼音搜索
- 搜索范围：所有层级（省名、市名、区县名）
- 输入时隐藏下方列表，展示搜索结果
- 搜索结果显示完整路径（如「广西 > 南宁 > 青秀区」）

#### 4. 最近使用

- 展示最近 5 个选过的位置（显示完整路径）
- 首次无记录时隐藏
- 数据存储在 LocalStore 中

#### 5. 热门城市

- 保留 12 个热门城市快捷入口
- 点击后直接选择该城市（市级），不需要再选区县

#### 6. 三级下钻列表

交互方式为逐级下钻：

**第一级 — 省份列表：**
- 34 个省级行政区按拼音排列
- 右侧拼音字母索引条（A-Z）
- 直辖市（北京/天津/上海/重庆）和特别行政区也在此级

**第二级 — 城市列表：**
- 点击省份后显示该省下属地级市
- 顶部面包屑：「广西 >」（可点击回退）
- 列表按拼音排列

**第三级 — 区县列表：**
- 点击城市后显示该市下属区/县
- 顶部面包屑：「广西 > 南宁 >」（每级可点击回退）
- 点击区县完成选择，返回首页

**面包屑导航：**
- 位于搜索框下方
- 显示当前路径，每级可点击回退
- 格式：省份 > 城市 > （当前选择级）

### 数据方案

#### 数据文件

创建 `assets/china_divisions.json`，包含完整的中国行政区划数据：

```json
[
  {
    "name": "北京市",
    "pinyin": "beijing",
    "children": [
      {
        "name": "北京市",
        "pinyin": "beijing",
        "children": [
          {"name": "东城区", "pinyin": "dongcheng"},
          {"name": "西城区", "pinyin": "xicheng"},
          ...
        ]
      }
    ]
  },
  ...
]
```

三级结构：省 -> 市 -> 区/县。每个节点有 name 和 pinyin 字段。

#### GPS 匹配

保留 Haversine 方案，但改为匹配区县级别。每个区县节点增加 lat/lng 字段（仅用于 GPS 匹配的部分城市，不需要所有区县都有坐标）。

简化方案：GPS 匹配到市级即可（使用地级市中心坐标），用户可手动选择区县。

### LocalStore 变更

城市存储从单个字符串改为完整路径：

```dart
// 存储格式改为 "省份/城市/区县"
static String get city => ...;  // 保持向后兼容，返回显示名
static String get cityPath => ...; // "广西/南宁/青秀区"
static List<String> get cityPathParts => ...; // ["广西", "南宁", "青秀区"]

// 最近使用记录存完整路径
static List<String> get recentCities => ...; // ["广西/南宁/青秀区", "广东/深圳/南山区", ...]
```

### 搜索交互

- 搜索所有层级的名称和拼音
- 结果按匹配度排序：完全匹配 > 前缀匹配 > 包含匹配
- 每条结果显示完整路径，点击直接选择
- 无结果时显示提示

### 不变的部分

- 路由：`/city-picker`
- 首页 `_TopBar` 的城市展示（显示名称改为 "市" 或 "区" 级别）
- `cityProvider` 的响应机制
